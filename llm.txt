# circuit-synth

> Define circuits in Python; generate KiCad projects (schematic + PCB).

This is a **development fork** of circuit-synth, not the published PyPI package.
Install it locally (editable) in this workspace; the CLI entry points below come
from that install. The API examples in this file are verified against the code
in `src/circuit_synth/`.

## What it does

Turn a Python circuit definition into a complete KiCad project:

```python
from circuit_synth import circuit, Component, Net

@circuit(name="Power_Supply")
def power_supply():
    vreg = Component(
        symbol="Regulator_Linear:AMS1117-3.3",
        ref="U",
        footprint="Package_TO_SOT_SMD:SOT-223-3_TabPin2",
    )
    cap_in = Component(symbol="Device:C", ref="C", value="10uF")
    cap_out = Component(symbol="Device:C", ref="C", value="22uF")

    vbus = Net("VBUS")
    vcc_3v3 = Net("VCC_3V3")
    gnd = Net("GND")

    vreg["VI"] += vbus
    vreg["VO"] += vcc_3v3
    vreg["GND"] += gnd
    cap_in[1] += vbus
    cap_in[2] += gnd
    cap_out[1] += vcc_3v3
    cap_out[2] += gnd

if __name__ == "__main__":
    c = power_supply()
    c.generate_kicad_project("my_power_supply")
```

Result: a KiCad project directory with `.kicad_sch` and `.kicad_pcb` files.

## Core API

Exported from `circuit_synth`: `circuit`, `Component`, `Net`, `Pin`, `Circuit`.

### Components

```python
from circuit_synth import Component

component = Component(
    symbol="Library:SymbolName",        # KiCad symbol path (required)
    ref="U",                            # reference prefix (U, R, C, ...)
    footprint="Library:FootprintName",  # KiCad footprint path (optional)
    value="10k",                        # optional value
)

# Any extra keyword becomes a component field (e.g. for BOM / sourcing)
r = Component(
    symbol="Device:R", ref="R", value="10k",
    footprint="Resistor_SMD:R_0603_1608Metric",
    manufacturer="Yageo", part_number="RC0603FR-0710KL", tolerance="1%",
)
```

### Nets

```python
from circuit_synth import Net

vcc = Net("VCC_3V3")
component["VCC"] += vcc   # named pin
component[1]     += vcc   # numbered pin
# Connect several pins to the same net by adding each to it.
```

### Circuit decorator and hierarchy

```python
from circuit_synth import circuit

@circuit(name="Power_Module")
def power_module(vin, vout, gnd):
    """Docstring becomes a schematic annotation."""
    vreg = Component(symbol="Regulator_Linear:AMS1117-3.3", ref="U")
    vreg["VI"] += vin
    vreg["VO"] += vout
    vreg["GND"] += gnd

@circuit(name="Main")
def main():
    vbus, vcc_3v3, gnd = Net("VBUS"), Net("VCC_3V3"), Net("GND")
    power_module(vbus, vcc_3v3, gnd)   # subcircuits are just called as functions
    led = Component(symbol="Device:LED", ref="D")
    led["A"] += vcc_3v3
    led["K"] += gnd
```

## Generation and manufacturing outputs

All are methods on the circuit object returned by a `@circuit` function.

```python
c = main()

# Full KiCad project (schematic + PCB). Key kwargs shown with their defaults:
c.generate_kicad_project(
    project_name="my_board",
    generate_pcb=True,
    placement_algorithm="hierarchical",   # or "simple"
    force_regenerate=False,               # True discards manual KiCad edits
    update_source_refs=None,              # see "Reference rewriting" below
    # erc_gate=True and selective_wires=True are on by default in this fork
)

c.generate_bom(project_name="my_board")             # -> my_board/... BOM (CSV)
c.generate_pdf_schematic(project_name="my_board")   # -> PDF schematic
c.generate_gerbers(project_name="my_board")         # -> Gerber/drill files
```

`generate_bom`/`generate_pdf_schematic`/`generate_gerbers` also accept
`output_file`/`output_dir` and format options (e.g. `exclude_dnp`,
`black_and_white`, `drill_format`). Export uses KiCad's `kicad-cli`.

### Reference rewriting

When refs are finalized during generation (e.g. `ref="C"` becomes `ref="C1"`),
circuit-synth can write the finalized references back into your Python source
file. This is controlled by `update_source_refs` (default `None` = auto-update
unless `force_regenerate=True`; pass `True`/`False` to force the behavior).

### Bidirectional / round-trip

Python Ôćĺ KiCad and KiCad Ôćĺ Python are both supported (entry points
`python-to-kicad` and `kicad-to-python`), so you can edit in either and re-sync.

## Project layout

A project is a flat directory of Python files ÔÇö a top-level `main.py` that
imports subcircuit modules:

```
my_project/
ÔöťÔöÇÔöÇ main.py            # top-level circuit; imports and uses subcircuits
ÔöťÔöÇÔöÇ power_supply.py    # a subcircuit
ÔööÔöÇÔöÇ usb_interface.py   # a subcircuit
```

```python
# main.py
from circuit_synth import circuit, Net
from power_supply import power_supply
from usb_interface import usb_c_interface

@circuit(name="Main_Board")
def main_board():
    vbus, vcc_3v3, gnd = Net("VBUS"), Net("VCC_3V3"), Net("GND")
    usb_c_interface(vbus, gnd)
    power_supply(vbus, vcc_3v3, gnd)

if __name__ == "__main__":
    main_board().generate_kicad_project("main_board")
```

## Common example patterns

### LED with current-limiting resistor

```python
@circuit(name="LED_Indicator")
def led_indicator():
    led = Component(symbol="Device:LED", ref="D",
                    footprint="LED_SMD:LED_0603_1608Metric")
    r = Component(symbol="Device:R", ref="R", value="330",
                  footprint="Resistor_SMD:R_0603_1608Metric")
    vcc, gnd, anode = Net("VCC_3V3"), Net("GND"), Net("LED_ANODE")
    r[1] += vcc
    r[2] += anode
    led["A"] += anode
    led["K"] += gnd
```

### USB-C with CC resistors

```python
@circuit(name="USB_C_Interface")
def usb_c_interface():
    j = Component(symbol="Connector:USB_C_Receptacle_USB2.0", ref="J",
                  footprint="Connector_USB:USB_C_Receptacle_GCT_USB4085")
    cc1_r = Component(symbol="Device:R", ref="R", value="5.1k")
    cc2_r = Component(symbol="Device:R", ref="R", value="5.1k")
    vbus, gnd, cc1, cc2 = Net("VBUS"), Net("GND"), Net("CC1"), Net("CC2")
    j["VBUS"] += vbus
    j["GND"]  += gnd
    j["CC1"]  += cc1
    j["CC2"]  += cc2
    cc1_r[1] += cc1; cc1_r[2] += gnd
    cc2_r[1] += cc2; cc2_r[2] += gnd
```

More complete, runnable examples live in
`src/circuit_synth/data/templates/example_circuits/`:
`power_supply_module.py`, `usb_c_basic.py`, `esp32_dev_board.py`,
`stm32_minimal.py`.

## Common symbols and footprints

Stable, commonly used KiCad identifiers:

- Passives: `Device:R`, `Device:C`, `Device:L`, `Device:LED`
- Connectors: `Connector:USB_C_Receptacle_USB2.0`,
  `Connector_Generic:Conn_01x08_Pin`, `Connector:Barrel_Jack_Switch`
- Regulators: `Regulator_Linear:AMS1117-3.3`, `Regulator_Linear:AMS1117-5.0`
- Modules: `RF_Module:ESP32-C6-MINI-1`, `MCU_ST_STM32F4:STM32F407VETx`
- Footprints: `Resistor_SMD:R_0603_1608Metric`,
  `Capacitor_SMD:C_0805_2012Metric`, `Package_TO_SOT_SMD:SOT-223-3_TabPin2`,
  `Package_QFP:LQFP-64_10x10mm_P0.5mm`

## Verify symbols and footprints ÔÇö don't trust a static list

Symbol/footprint names vary by KiCad version, and an LLM can invent names that
don't exist. **Do not rely on a hardcoded "known-good" list.** Verify against
the installed libraries before using an unfamiliar identifier.

Generated projects include `tools/find_symbol.py` for this; the `design-circuit`
skill drives symbol/footprint lookup as part of the design loop. Use those
rather than guessing.

## Skills and tooling

The design loop is driven by skills and tools, not by ad-hoc slash commands:

- `new-kicad-project` skill ÔÇö bootstrap a new project (`cs-new-project` /
  `cs-bootstrap` CLI entry points).
- `design-circuit` skill ÔÇö the design Ôćĺ verify Ôćĺ generate loop, including
  symbol/footprint lookup via `tools/find_symbol.py`.
- kicad-sch-api MCP server ÔÇö programmatic read/write of `.kicad_sch` files.

## Troubleshooting

- Generation runs for ~10ÔÇô30s with verbose output; this is normal.
- Files are written in order: `.kicad_pro`, then `.kicad_sch`, then `.kicad_pcb`.
  If only `.kicad_pro` exists, the run was interrupted ÔÇö re-run it.
- `WARNING - Reached maximum placement attempts (100)` is non-fatal; components
  are still placed. Adjust positions in KiCad if needed. Let the run finish.

## Architecture

JSON is the canonical intermediate representation:

```
Python circuit Ôćĺ JSON Ôćĺ KiCad files (.kicad_sch, .kicad_pcb)
                   ÔćĹ                        Ôćô
                   ÔööÔöÇÔöÇÔöÇÔöÇÔöÇÔöÇÔöÇÔöÇ round-trip ÔöÇÔöÇÔöÇÔöÇÔöś
```

This gives a text-based, version-controllable, tool-agnostic source of truth.
See `docs/ARCHITECTURE.md` and `docs/JSON_SCHEMA.md`.
